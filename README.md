# trading-bot (draft)

한국투자증권(KIS) Developers API로 붙는 트레이딩 봇 초안. 특정 종목 하나를 보는 게
아니라, KIS 거래량순위 API로 후보 종목 리스트를 받아오고, 뉴스 감성과 SMA 크로스를
합쳐 기댓값이 가장 높은 종목 하나를 골라 투자함. WinHTTP(Windows 내장)로 HTTPS
처리해서 libcurl 등 외부 의존성 없음. JSON은 nlohmann/json 헤더 하나만 vendor.

## 실행 모드 (`config.json`의 `mode`)

| mode | 시세/종목리스트 | 주문 | 필요한 것 |
|---|---|---|---|
| `mock` (기본) | 로컬 가상 종목 5개, 랜덤워크로 합성 | 로컬 가상 체결 | 없음 — 네트워크/계좌 전부 불필요 (뉴스 크롤링도 스킵) |
| `sim` | KIS 실계좌 앱키로 실시간 거래량순위+시세 | 로컬 가상 체결 (KIS에 주문 안 보냄) | 실계좌 App Key만 (모의투자 신청 불필요) |
| `paper` | KIS 모의투자 서버 | KIS 모의투자 주문 | 모의투자 신청 완료 + 모의투자 계좌 |
| `live` | KIS 실서버 | **진짜 돈** 주문 | 실계좌, 신중하게 |

`sim`이 가장 현실적인 테스트 방법: 실제 거래량순위/시세로 전략을 검증하면서도 KIS의
모의투자 신청 절차나 가상계좌 설정을 거칠 필요가 없음 — 주문 자체를 로컬에서만 가상
체결(`sim_fills.log`에 기록)하고 KIS 주문 API는 아예 호출하지 않음.

## 동작

1. **종목 리스트 조회**: 보유 중인 종목이 없으면 KIS 거래량순위 API로 상위
   `watchlist_size`개 후보를 받아옴 (레버리지/인버스 ETF·ETN은 자동 제외 — 시간가치
   감소가 있고 변동성도 커서 단순 SMA 크로스 전략에 안 맞음).
2. **뉴스 크롤링**: 네이버/다음이 아닌 매일경제·한국경제·연합뉴스 증권/경제 RSS
   피드를 매 스캔마다 한 번씩 받아옴 (`news_feeds`로 설정 가능). 유튜브/SNS는
   포함하지 않음 — 공식 API 인증(YouTube Data API, X API 등)이 필요하고 비공식
   스크래핑은 차단/불안정 위험이 커서 이번 단계에서는 제외.
3. **영향 계산**: 각 후보 종목명이 언급된 기사 제목/본문에서 긍정/부정 키워드
   사전으로 점수화 (`NewsCrawler::scoreSentiment`) — 정교한 NLP가 아니라 키워드
   매칭 기반의 거친 추정치.
4. **기댓값 계산 및 매수**: SMA 골든크로스가 뜨는 후보 중, 기댓값(`얻을 이득 ×
   얻을 확률`)이 가장 높은 종목 하나를 시장가로 매수. 이득은 `take_profit_pct`
   기준 목표 수익, 확률은 뉴스 감성 점수를 0.05~0.95 사이로 매핑한 값 — 이 확률은
   백테스트로 보정된 값이 아니라 휴리스틱임.
5. **매도**: 보유 중 데드크로스가 뜨거나, 수수료(`fee_rate`, 매수/매도 양쪽)와
   세금(`tax_rate`, 매도 시)을 뺀 순손익률이 `take_profit_pct` 이상이면 시장가로
   매도 (둘 중 먼저 뜨는 조건). 데드크로스=손절/추세이탈, 수익실현=목표 도달, 두
   조건 다 사용.
6. **보유 현황 파일**: `holdings.txt`를 5초마다 갱신 (poll_seconds와 무관하게
   별도 스레드가 돌림) — 지금 뭘 들고 있는지 파일만 보면 항상 알 수 있음.
7. **거래 로그**: 매수/매도할 때마다 콘솔+`trading.log`에 사람이 읽는 로그를 남기고,
   `trades.log`에 `시각,BUY/SELL,코드,종목명,수량,가격,수수료율,세율,순손익` 형태로
   한 줄씩 남김.

`poll_seconds`마다 반복.

## 준비

- `mock`: 아무 준비 없이 바로 실행 가능.
- `sim`: https://apiportal.koreainvestment.com 가입 → 앱 등록 → App Key/App Secret 발급
  (실계좌 연결, 모의투자 신청은 불필요).
- `paper`/`live`: 위 App Key/Secret에 더해 계좌번호(`cano`, `acnt_prdt_cd`) 필요.
  `paper`는 모의투자 신청까지 완료해야 함.

`config.example.json`을 `config.json`으로 복사해서 값 채우기. `config.json`은
`.gitignore`에 이미 등록되어 있음. 주요 파라미터:

| 필드 | 의미 | 기본값 |
|---|---|---|
| `watchlist_size` | 매 스캔마다 볼 후보 종목 수 | 5 |
| `fee_rate` | 매매수수료 (매수/매도 양쪽) | 0.00015 (0.015%) |
| `tax_rate` | 증권거래세 (매도 시) | 0.0018 (0.18%) |
| `take_profit_pct` | 이 이상 순손익률이면 매도 | 0.02 (2%) |
| `news_feeds` | 크롤링할 RSS 피드 URL 목록 | 매경/한경/연합뉴스 |

`fee_rate`/`tax_rate`는 시기·증권사·상품에 따라 실제 값이 다르니 사용 전에 KIS/증권사
고시 요율로 확인할 것 — 여기 기본값은 대략적인 참고치임.

## 빌드 (MSYS2 UCRT64 g++ 또는 MSVC)

```
cmake -S . -B build
cmake --build build
```

`build/` 안에 `config.json`을 두고 실행:

```
cd build
./trading_bot.exe
```

## 테스트

전략 로직(SMA 크로스/모멘텀/순손익률)과 `MockBroker`, `NewsCrawler` 감성 점수를
결정론적으로 검증하는 self-check:

```
cmake --build build --target test_strategy
./build/test_strategy.exe
```

## 실시간 확인

- 콘솔(및 `trading.log`)에 스캔마다 후보 종목 각각의
  `종목명(코드) 현재가=... 시그널=... 뉴스감성=...`가 찍히고, 매수/매도가 체결되면
  `>>> 매수 체결 ...` / `<<< 매도 체결 ... 순손익 ...` 줄이 추가로 찍힘.
- `holdings.txt`: 지금 보유 중인 종목/수량/평단가/평가손익을 5초마다 갱신.
- `trades.log`: 체결마다 한 줄씩, 수수료/세금 반영한 순손익 포함.
- 콘솔 한글이 깨지면 터미널 자체가 아니라 프로그램이 `SetConsoleOutputCP(CP_UTF8)`로
  맞춰두니, 그래도 깨지면 터미널 폰트/코드페이지 문제일 가능성이 높음.

## 검증 필요

- 실제 KIS 모의투자 서버로 인증·거래량순위·시세 조회·뉴스 크롤링(220건 수집,
  삼성전자 감성점수 산출 확인)·매수/매도 체결까지 전부 확인 완료 (2026-07-22).
- 기댓값의 "확률"은 뉴스 감성 점수를 선형으로 매핑한 휴리스틱일 뿐, 실제 상승 확률을
  통계적으로 추정한 값이 아님 — 실거래 투입 전에 백테스트로 검증 필요.
- 뉴스 크롤링은 RSS 헤드라인/설명에 종목명이 문자열로 포함되는지만 봄 — 별명/약칭
  (예: "삼전")은 못 잡음, 뉴스가 없으면 감성점수는 항상 0.
- KIS는 초당 호출 횟수 제한이 빡빡함(모의투자 계좌에서 `EGW00201` 레이트리밋을 실제로
  겪음). API 호출 사이에 1.1초 간격을 둠. 거래량순위 응답에 종목별 현재가
  (`stck_prpr`)가 이미 포함돼 있어서 후보마다 따로 현재가 조회를 하지 않음 — 스캔당
  호출은 순위조회 1번 + 후보당 일봉조회 1번뿐. mock 모드는 실제 API가 아니라서 이
  간격이 없음.
- 매수 후 모니터링용 하루봉 히스토리는 매수 시점에 한 번만 캐시함 — 포지션을 자정
  넘어 여러 날 들고 있으면 히스토리가 하루치 갱신이 안 되는 게 알려진 한계.
- ETF/ETN 필터는 이름에 `KODEX`/`TIGER`/`레버리지`/`인버스` 등 문자열이 포함되는지로
  판단하는 간단한 휴리스틱 — 모든 상품군을 완벽히 걸러내진 못함.
- tr_id, 필드명 등은 문서 기준으로 작성했지만 API가 종종 바뀌므로 실제 응답으로
  재확인할 것: https://apiportal.koreainvestment.com
- 장 운영시간(09:00~15:30) 체크, 잔고 조회 로직은 아직 없음 — `sim`/`paper` 모드로
  먼저 충분히 굴려보고 필요한 부분만 추가.
- `mode: "live"`로 바꾸면 진짜 돈으로 주문이 나감. 절대 검증 없이 바로 바꾸지 말 것.
